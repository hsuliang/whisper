# 邱老師的 Claude Code 使用重點提醒

> 適用於：Claude Code Desktop / claude.ai
> 作者：邱老師（文盛老師）
> 更新：2026-04

---

## 🔴 層次一：省錢（Token 用量管理）

### 先規劃再執行
- ✅ 永遠先用 Plan Mode 確認方向，再開始寫程式
- ❌ 不要直接說「幫我做 XXX」→ 做錯 → 重做（浪費 50~70% Token）

### 一次說清楚
- ✅ 把風格、語言、限制、功能一次說完
- ❌ 不要分多次補充需求（每次補充都耗 Token）

### 善用「繼續」
- Claude 停下來時，直接說「繼續」或「完成剩下的部分」
- 不要重新貼一次需求（會重複計算 Token）

### CLAUDE.md 是最划算的投資
- 只需要寫一次，每個 Session 自動載入
- 把偏好、限制、風格全寫進去，省掉每次重新解釋

## 🟡 層次二：省力（記憶與上下文管理）

### Session 變長時壓縮記憶
貼入以下指令讓 Claude 摘要狀態：
```
請摘要目前專案狀態：已完成、進行中、待處理、重要決定，
用條列式整理並更新到 CLAUDE.md 的「目前狀態」區塊。
```

### CLAUDE.md 要有「狀態區塊」
每個專案的 CLAUDE.md 最後加：
```markdown
## 目前開發狀態
- 完成：
- 進行中：
- 待處理：

## 重要決定記錄
-
```

### 全域記憶（Claude Code 專用）
- 在終端機輸入 `/memory` 可查看與編輯全域記憶
- 適合儲存：個人偏好、慣用風格、跨專案設定

## 🟢 層次三：升級（讓 Claude 越用越聰明）

### 建立提示詞庫
每次做完專案，把成功的 Prompt 存到「我的提示詞庫.md」：
- 抽獎系統類
- 教育表單類
- 成績計算類
- ……持續累積

### 角色設定（寫入 CLAUDE.md）
```markdown
## Claude 的角色設定
你是專為台灣國中小教師服務的前端工程師：
- 所有介面使用繁體中文
- 風格活潑適合教育現場
- 程式碼要有中文註解
- 優先考慮手機操作便利性
```

### 完成後請 Claude 自我檢討
每個專案結束後輸入：
```
請檢視這個專案，列出：
1. 哪些地方可以優化
2. 有沒有潛在的 bug
3. 對下一個類似專案的建議
```

## ⚡ 速查清單

| 時機 | 動作 |
|------|------|
| 開新專案前 | 更新 CLAUDE.md，寫清楚需求與限制 |
| 開始寫程式前 | Plan Mode 先確認架構 |
| Session 變長時 | 請 Claude 摘要存入 CLAUDE.md |
| 專案完成後 | 請 Claude 自我檢討，存好提示詞 |
| 下次新 Session | 確認 CLAUDE.md 是最新狀態再開始 |
